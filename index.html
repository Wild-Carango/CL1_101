CLI 101 / Overview

Welcome to CLI 101

🎯 What You'll Learn

This course covers four essential command-line tools for Windows users. By the end, you'll know what each tool is for, their basic commands, syntax patterns, and when to use which one. Navigate using the tabs above or sidebar — start wherever you like, or follow in order.

Microsoft's modern scripting shell. More powerful than CMD, with object-based pipelines and deep Windows integration.

How They Relate

WSL

A compatibility layer. It's a bridge — installs Ubuntu (or other distros) so you can run Linux tools on Windows. WSL contains Ubuntu CLI.

Ubuntu CLI

The actual shell you use inside WSL. Commands here are Linux Bash commands. Think of Ubuntu as the "OS inside WSL".

CMD

Completely separate from WSL/Ubuntu. Native Windows. Runs .bat scripts, older Windows utilities. Limited but simple.

PowerShell

Also native Windows. Newer, more powerful than CMD. Runs .ps1 scripts. Great for automation and system administration.

How to Open Each One

Windows — How to Launch
# WSL — open from Start Menu, search "WSL" or "Ubuntu"
# Or from any terminal: type wsl and press Enter
 
# CMD — press Win+R, type cmd, press Enter
# Or search "Command Prompt" in Start Menu
 
# PowerShell — press Win+X, select "PowerShell" or "Terminal"
# Or search "PowerShell" in Start Menu
 
# Windows Terminal — best app: runs WSL, CMD, PowerShell in tabs
# Install from Microsoft Store (search "Windows Terminal")

💡 Pro Tip: Use Windows Terminal

Install Windows Terminal from the Microsoft Store. It lets you open WSL, CMD, and PowerShell as tabs in one window — the best way to learn all four tools side by side.

CLI 101 / WSL

WSL — Windows Subsystem for Linux

What is WSL?

WSL is a feature built into Windows 10/11 that lets you run a full Linux environment — including a shell, file system, and command-line tools — directly on Windows. No virtual machine, no rebooting. WSL 2 (the current version) uses a real Linux kernel for maximum compatibility.

Why Use WSL?

  • Run Linux tools (grep, awk, sed, bash scripts) on Windows
  • Web developers can use the same environment as their Linux servers
  • Access Windows files from Linux and vice versa
  • Use package managers like apt to install software
  • Much faster and lighter than a full virtual machine

Installation Commands (run in PowerShell as Admin)

PowerShell — WSL Installation
# Install WSL + default Ubuntu in one command (Windows 11 / Win10 updated)
PS C:\> wsl --install
 
# List available distros you can install
PS C:\> wsl --list --online
 
# Install a specific distro (e.g., Ubuntu-22.04)
PS C:\> wsl --install -d Ubuntu-22.04
 
# Check your installed distros
PS C:\> wsl --list --verbose
 
# Check WSL version
PS C:\> wsl --version
 
# Set WSL 2 as default
PS C:\> wsl --set-default-version 2
 
# Launch WSL (opens your default distro)
PS C:\> wsl
 
# Shut down all WSL instances
PS C:\> wsl --shutdown

Accessing Files Between Windows & Linux

File System Access
# From INSIDE WSL — access Windows C: drive
user@machine:~$ cd /mnt/c/Users/YourName/Desktop
 
# From WINDOWS Explorer — access Linux files
# Type this in Explorer address bar:
\\wsl$\Ubuntu\home\yourusername
 
# Open current Linux folder in Windows Explorer
user@machine:~$ explorer.exe .
 
# Open a Windows app from WSL (e.g., VS Code)
user@machine:~$ code .

💡 WSL vs WSL2

Always use WSL 2 — it has a real Linux kernel, better file system performance, and supports Docker. Run wsl --set-default-version 2 in PowerShell to make it the default.

CLI 101 / Ubuntu CLI

Ubuntu CLI — Bash Shell

What is the Ubuntu CLI?

Ubuntu's command-line interface uses Bash (Bourne Again Shell) — the default shell on most Linux systems. When you open WSL, you're dropped into a Bash terminal. This is the same environment professional developers and sysadmins use on servers around the world.

The Prompt Explained

Ubuntu Bash
username@hostname:~$ ← your prompt
# username = your Linux user
# hostname = your computer name
# ~ = current directory (~ means your home folder)
# $ = regular user | # = root/admin user

⚠️ Warning: rm -rf is dangerous

The command rm -rf permanently deletes files with no recycle bin. Always double-check before using it. Never run sudo rm -rf / — that would erase your entire Linux system.

CLI 101 / CMD — Command Prompt

CMD — Windows Command Prompt

What is CMD?

Command Prompt (cmd.exe) is Windows' original command-line shell, dating back to DOS. It's simpler and less powerful than PowerShell, but it's still on every Windows computer and is great for quick tasks, running legacy batch files, and basic file/network operations.

The Prompt Explained

CMD

# C: = current drive letter
# \Users\YourName = current folder path
# > = ready for your command

CLI 101 / PowerShell

PowerShell

What is PowerShell?

PowerShell is Microsoft's modern, powerful scripting shell. Unlike CMD, which passes text between commands, PowerShell passes objects — structured data with properties and methods. It's deeply integrated with Windows, .NET, and Microsoft services, making it the go-to tool for IT admins and automation.

The Prompt Explained

PowerShell
PS C:\Users\YourName> ← PS prompt
# PS = PowerShell indicator
# C:\Users\YourName = current directory
# > = ready for input

Navigation & File Commands (Cmdlets)

💡 Cmdlet Naming Pattern

PowerShell commands are called cmdlets (pronounced "command-lets"). They follow a Verb-Noun pattern: Get-Item, Set-Location, New-Item. Many Linux-like aliases also work: ls, cd, pwd, cat.

Move-Item / mvMove or renameMove-Item old.txt new.txt
Get-Content / catRead file contentsGet-Content notes.txt

PatternMeaningExample

PowerShell — Real Examples
# List only .txt files in current folder
PS C:\> Get-ChildItem *.txt
 
# Find top 5 CPU-hungry processes
PS C:\> Get-Process | Sort-Object CPU -Descending | Select-Object -First 5
 
# Create a folder and file in one line
PS C:\> New-Item -Path "C:\test\myfile.txt" -ItemType File -Force
 
# Get your public IP address
PS C:\> (Invoke-WebRequest -Uri "https://api.ipify.org").Content
 
# Simple loop
PS C:\> 1..5 | ForEach-Object { Write-Output "Line $_" }
